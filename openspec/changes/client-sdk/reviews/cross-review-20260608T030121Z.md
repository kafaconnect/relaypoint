- **When:** 20260608T030121Z
- **Builder:** claude (+ delegated authoring subagent)
- **Independent reviewers:** codex (read-only sandbox), agy (print mode)
- **Scope:** openspec/changes/client-sdk/{proposal,design,tasks,specs} + signaling-core refinements
- **Rubric:** cross-review skill R1..R8 (enumerate the negative space)

## Consolidated verdict: BLOCKED (both reviewers)

### Confirmed BLOCKERs (union)
1. signaling-core still specifies **warm transfer in M1** (scenario `signaling.call.transfer`) — contradicts locked C1 cold-only. [codex] (+ user: transfer is mis-placed on CallController)
2. **Recording fact vocabulary drift** — core has only `recording.started/stopped`; SDK needs consent requested/granted/denied + retention ref + recorder identity + upload status + failure. [codex+agy]
3. **Command idempotency/ack unspecified** — `.cmd` has no command_id/ack; a client retry can double-append router facts. [codex]
4. **SDK state machines not specified** (connection/call/interaction/recording) — only happy paths; interfaces expose no way to READ call/interaction state. [codex+agy] (+ user: call state + video not observable)

### HIGH
- MediaAdapter has createOffer/applyRemote but no answer/renegotiation descriptor path. [codex]
- Recording capability conflates local best-effort with server/compliance (startLocalRecording gated wrong). [codex]
- Media-IAM down / ticket expiry / credential refresh / typed errors unspecified. [codex+agy]
- getToken() failure unspecified. [agy]

### MEDIUM
- Offer control handling omits terminals timed_out_rona/expired/no_responder_fast_rona. [codex]
- LogEvent omits schema/tenantId though 'verbatim envelope'. [codex]
- OfferTerminal `expired` has no scenario vs timed_out_rona. [agy]
- signaling-core tasks.md envelope line omits `media_profile?`. [codex]

### User-raised (during read; fold into the same revision)
- A. `transfer` belongs on InteractionHandle, not CallController (routing, not media).
- B. Video call not represented: media kind (audio/video), track/render exposure, mute, state observability.
- C. Interaction always carries **metadata** (customer info, integration data, enriched over time) — not represented.

### Open design forks (need a decision before revision)
- Metadata transport: opaque `context` blob on the interaction log (router-ordered, observable) **vs** SDK fetch from Desk API (relaypoint unaware).
- Command idempotency: in the universal signaling-core command envelope (router dedups) **vs** SDK retry policy only. [codex Q]
- relaypoint-go offer publish: via the router (router owns ring + state machine) **vs** direct publish to routing.offer.user.<target>. [codex Q]

### DECISION_FIDELITY
B1/B3 creds, non-writer .log, own-author signal, opaque media_profile, two-SDK split = faithful. **C1 not faithful in signaling-core** (warm transfer remains). **D not faithful in signaling-core** (recording vocab missing).
