# Adversarial AI debate — S3 (signaling-core + client-sdk + nats-deploy)

A two-round "cãi nhau" review: each reviewer (codex, agy) filed opening claims, then
cross-examined the OTHER's claims (REFUTE / CONCEDE / ESCALATE) with code evidence. Builder
(claude) adjudicated. Surface: review/s3-integration with the prior deep-review fixes applied.

## Round 2 outcome: 0 REFUTE — every claim CONCEDED or ESCALATED. All survived → all fixed.

| Claim | Round-2 verdict | Defect | Fix |
|---|---|---|---|
| event_id counter (X5/A6) | CONCEDE | process-local `ev-N` resets on restart → collides with ids in the durable log | random crypto id default (signaling-core) |
| stale seq (A1/X4) | **ESCALATE→BLOCKER** | dup/append-fail left in-memory `seq` stale for goroutines already holding the state; ack-loss reported as rejected | `poisoned` flag checked after lock + evict; reconcile-from-log on append error → accept if committed (signaling-core, TestCore_DupRebuildFailEvicts) |
| cross-restart conflict (X1) | CONCEDE/BLOCKER | rebuilt facts had no payload hash → divergent command_id reuse replayed accepted | facts carry `payload_hash`; rebuild populates it (signaling-core, TestCore_ConflictAcrossRestart) |
| Mailbox lock (A3) | CONCEDE | single-consumer flag had no `finally` → a broken loop permanently locked events() | release on stop via try/finally (client-sdk, test) |
| close-leak (X3/A4) | CONCEDE | close mid-connect guarded state but leaked the late connection | connect()/reconnect() close the late conn when closed (client-sdk, test) |
| handle reuse (A5) | CONCEDE | closed handles cached + resubscribed on reconnect | close() drops from client cache; interaction() re-creates (client-sdk, test) |
| replay ACL (X2) | CONCEDE/BLOCKER | replay did $JS.API.STREAM.NAMES discovery the client ACL forbids | bindStream("INTERACTION_LOGS") — only $JS.API.CONSUMER.> needed (client-sdk) |
| replay hang/leak (A2) | CONCEDE | ordered replay blocked forever on an empty stream + leaked the consumer on close | terminate on stream-head / inactivity / close (client-sdk) |
| docs drift (X7) | CONCEDE | client-sdk spec + arch HTML still said `Nats-Msg-Id = event_id` | corrected to the command-derived publish key (signaling-core) |
| ACL scenarios (X6) | CONCEDE | tenant-isolation / issuer-inbox scenarios presented as satisfied | marked explicitly NOT enforced in Phase-1, pending auth-callout (signaling-core) |

## Follow-up (tracked, not silently dropped)
- The nats.ws adapter replay (bindStream + inactivity termination) is NOT exercisable by the
  fake-transport unit suite — it needs a LIVE-NATS integration test under the deployed client
  ACL before the browser replay path is trusted (codex X2 raised this explicitly).
- The "state unavailable — retry" reply on a poisoned append is a `rejected` status; the SDK
  does not yet auto-retry rejected results — wiring a retryable signal is a follow-up.
